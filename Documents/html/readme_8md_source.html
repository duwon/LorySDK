17 ## Introduction
18 
19 The aim of this project is to show an example of the endpoint LoRaWAN stack implementation.
20 
21 This LoRaWAN stack implements all regions defined in "LoRaWAN Regional Parameters v1.0.2rB" document. Class A and Class C endpoint implementation
22 is fully compatible with "LoRaWAN specification 1.0.2".
23 
24 Each LoRaWAN application example includes the LoRaWAN certification protocol implementation.
25 
26 SX1272/76 radio drivers are also provided.
27 In case only point to point links are required a Ping-Pong application is provided as example.
28 
29 *The LoRaWAN stack API documentation can be found at: http://stackforce.github.io/LoRaMac-doc/*

438  **REMARK 1** *The regulatory duty cycle enforcement is enabled by default
439  which means that for lower data rates the node may not transmit a new
440  frame as quickly as requested.
441  The formula used to compute the node idle time is*
442 
443  *Toff = TimeOnAir / DutyCycle - TxTimeOnAir*
444 
445  *Example:*
446  *A device just transmitted a 0.5 s long frame on one default channel.
447  This channel is in a sub-band allowing 1% duty-cycle. Therefore this
448  whole sub-band (868 MHz - 868.6 MHz) will be unavailable for 49.5 s.*
449 
450  **REMARK 2** *The duty cycle enforcement can be disabled for test
451  purposes by calling the LoRaMacSetDutyCycleOn function with false
452  parameter.*

454  * Added a function to create new channels
